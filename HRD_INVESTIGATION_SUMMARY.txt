PROBLEM STATEMENT
-----------------
Need to control Yaesu FT-DX10 radio for SOTA operations:
- Set frequency (for tuning to SOTA spots)
- Set mode (USB/LSB/CW based on band)
- Log QSOs to Ham Radio Deluxe Logbook

PHASE 1: HRD TCP API INVESTIGATION
-----------------------------------
Protocol Details Discovered:
- Port: 7809 (localhost)
- Encoding: UTF-16LE
- Message format: 4-byte little-endian length prefix + UTF-16LE text
- Commands: "get context", "get frequency-hz", "set frequency-hz", "set dropdown mode"

Initial Results:
✅ Connection works
✅ Frequency reading works
✅ Frequency setting works
❌ Mode setting BROKEN (commands accepted but no effect)

CONCLUSION: HRD TCP API mode setting is fundamentally broken
- Not a syntax issue (other dropdowns work)
- Not a timing issue (tried delays, fresh context)
- Not an encoding issue (frequency setting works)
- API accepts commands but doesn't execute them
- This is a bug in HRD v6

PHASE 3: ALTERNATIVE SOLUTION - DIRECT CAT CONTROL
---------------------------------------------------
Decision: Bypass HRD for rig control, use direct serial CAT

COM Port Discovery:
- Used probe_alternatives.py to enumerate ports
- Found: COM7 (FT-DX10 serial interface)

Yaesu CAT Protocol:
- Baud: 9600
- Format: 8N2 (8 data bits, no parity, 2 stop bits)
- Commands: 5-byte ASCII terminated with semicolon
- Examples: FA; (read freq), MD1; (set USB), FA00007040000; (set 7.040 MHz)

CAT Implementation Scripts:
test_cat.py (7 versions) - Progressive testing and refinement
cat_client.py            - Production implementation

RESULTS:
✅ Direct CAT mode setting works perfectly
✅ Direct CAT frequency setting works
✅ All modes supported (LSB/USB/CW/FM/AM/RTTY/DATA/C4FM)
✅ Fast and reliable
✅ No dependency on HRD bugs

PHASE 4: HRD LOGBOOK INTEGRATION (UDP ADIF)
--------------------------------------------
Separate investigation for QSO logging functionality

Web Research Results:
- HRD Logbook accepts UDP ADIF on port 2333
- Same protocol used by WSJT-X, N1MM+
- Well-documented standard format

ADIF UDP Protocol:
- Port: 2333 (localhost)
- Format: ADIF records with field tags
- SOTA-specific fields: SIG="SOTA", SIG_INFO="W7W/LC-001"

Implementation:
- Added to bridge.py
- "Log QSO" button sends SOTA chase records
- Automatic field population from SOTAwatch

RESULTS:
✅ QSO logging works perfectly
✅ SOTA references properly encoded
✅ Appears in HRD Logbook immediately

LESSONS LEARNED
---------------
1. HRD TCP API is unreliable for write operations
   - Frequency setting works
   - Mode setting broken
   - Most other controls fail

2. Direct hardware control is superior
   - CAT protocol well-documented
   - Reliable and fast
   - No third-party software dependency

3. Industry standards work better
   - UDP ADIF is simple and reliable
   - Widely supported across ham radio software
   - Better than proprietary APIs

4. Systematic testing is essential
   - 13 probe scripts isolated the exact problem
   - Documented what works and what doesn't
   - Led to correct architectural decision

5. Community resources are invaluable
   - GitHub projects (MiniDeluxe, slice-master)
   - Archived forum posts
   - Reverse engineering blog posts
   - WSJT-X source code
