================================================================================
LESSONS LEARNED
================================================================================

1. Systematic Testing is Essential
   - Created 9 probe scripts to isolate problem
   - Tested every possible variation
   - Documented negative results
   - Led to correct conclusion

2. Don't Trust Third-Party APIs for Critical Functions
   - HRD TCP API has bugs
   - Direct hardware control more reliable
   - Worth the extra effort

3. Industry Standards Work Better
   - UDP ADIF is well-documented
   - Widely supported across ham radio software
   - More reliable than proprietary APIs

4. When API Fails, Try Direct Control
   - Serial CAT protocol worked perfectly
   - No dependency on buggy software
   - Direct hardware communication is best

5. Archive Your Investigation
   - Kept all probe scripts
   - Documented what works and what doesn't
   - Valuable reference for future projects
